Purpose

This portfolio provides a look at a cross-section of the type of work I have completed, while serving as an example of my current level of proficiency with web technologies. The portfolio is intended to complement my resume which has more detail on past work experience.

Navigation

On wide format displays, you can select a project from the menu at left to explore it. On small format displays use the up and down arrow buttons to switch between projects. Access to project sections is achieved by clicking the appropriate tab on each project slide. This site also supports keyboard navigation; use left arrow/right arrow to move between sections, or i/j/k/l to jump between sections and projects.

Project Logistics

Some of the original concept work was completed through an individual assignment for an interaction design course at UC Berkeley in April 2010. The final product you're currently viewing was built in March, 2011. This is an open source project licensed under a Creative Commons Attribution 3.0 Unported License. The complete source code is available for forking on GitHub.

Planning involved identifying the audience, determining requirements, and sketching solutions.

Audience

My target user is a busy hiring manager or recruiter involved in screening candidates, potentially working on a mobile device. Goals of such a user include:

  • Understanding the type of projects I've worked on.
  • Determining my skill-level and ability with the technologies listed on my resume.
  • Finding material for discussion, or a reason to disqualify my candidacy.

Requirements

I decided my portfolio should show a cross-section of the type of work I've done while clearly explaining my role in the projects and the technologies used. There should be a manageable number of projects displayed, and several levels of detail available. Pages should load quickly and should accommodate different display sizes and interaction modalities.

Sketches

I tested a few potential designs through paper sketches.

I produced an initial high-fidelity prototype in HTML and CSS as part of the initial coursework. The result unearthed some usability issues that I took into account when building the final site.

The next step was to build out the view for a single project, making sure to handle presentation details in CSS. Once I was happy with the layout, I focused on making the interactive site flexible and fast

Building in Flexibility

I started with a jQuery base to gain more robust access to the DOM and for access to useful add-ons. I used the jQuery template plugin to create a skeleton that can be filled with project-specific content. With this structure in place it is trivial to make changes to all projects at once, or swap out one project for another. While not as powerful as a full MVC framework like Rails or Django, jQuery templates can run on any web server.

Making it Fast

I used a pair of jQuery plugins (jQuery UI and carouFredSel) for rich interactions that progressively reveal content and eliminate additional page loads. To speed up the perceived initial load time, I pushed actual content retrieval to a series of asynchronous requests. After the initial skeleton is built by javascript, each project is retrieved in order. This allows a visitor to get acquainted with the layout and start reading about the first project while the rest of the page finishes loading in the background. Finally, I was careful to optimize images to minimize file size, and to use Google and Microsoft hosted libraries when possible to take advantage of client-side caching.
