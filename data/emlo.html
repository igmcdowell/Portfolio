Emlo the "tangible broadcast buddy," is a prototype device for broadcasting your current emotion from a given location. User gestures are translated into a corresponding emotion, which is then posted (in combination with the user's location) to Twitter. Emlo was a class final project in collaboration with Alison Meir and Jessica Voytek.

Final Project Components (Creator in parenthesis):
  • Sensor Housing - The exterior surface of Emlo (Jessica, Alison)
  • Arduino Sensors and Code - The guts and brains of Emlo, built using an Arduino Duemilanove with flex sensor, force sensor, and photo sensor (Guts: Alison, Brains: Alison, Ian)
  • Python glue code - Local machine code for translating the Arduino serial input into HTTP Requests for the App Engine Back End. (Ian)
  • App Engine Back End - Used to store user messages, store per-user associations between gestures/locations and plain English names, reverse code geodata using the Google Maps API, call the Twitter API, and host the administrator front end. (Ian)
  • App Engine Front End - Administrator controls for managing descriptions of locations and gestures to be used in posted tweets. Also includes tool for generating fake tweets for demonstration purposes. (Ian)
  • Visual Assets - The project poster, clay prototypes, and digital renderings of Emlo (Jessica).
