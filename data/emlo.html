Emlo the "tangible broadcast buddy," is a prototype device for broadcasting your current emotion from a given location. User gestures are translated into a corresponding emotion, which is then posted (in combination with the user's location) to Twitter. Emlo was a class final project in collaboration with Alison Meir and Jessica Voytek.

Emlo was born out of an interest to explore gestural expression of emotion. After brainstorming different methods of exploring this question, we settled on the idea of a digital pet able to relay how its master is feeling. Given this goal, we set about exploring how individuals might express emotion. We started with a small number of exploratory interviews, and used the output to produce a close-ended survey. Finally, we determined what to build.

Interviews

We scoured a local drugstore for tangible artifacts with unique tactile or ballistic properties that could potentially be incorporated into a final project. We settled on a ping pong ball (bouncy, aerodynamic), sponge (squishy, deformable), hair scrunchy (elastic, deformable), and hair roller (spiny, rollable).

Six participants were individually interviewed and asked to model each of eight basic emotions using the provided materials. The gestures used were recorded and clustered by action. Ten distinct gestures were identified, which served as the basis for a follow-on survey.

Survey

We created a clay prototype and used it to demonstrate each gesture identified in the exploratory interviews in a series of short video clips. We incorporated these in an online survey asking users to map each of the eight emotions to one gesture (some gestures were left over). The 35 responses showed strongest agreement in gesture/emotion mappings for anger, sadness, joy, and surprise.

Determining What to Build

Hardware: The class was taught using Arduino, so we used a Duemilanove as the microcontroller. We assessed the top four emotion/gesture pairs for feasibility of implementation with off-the-shelf sensors, and determined we could model all four using less than $30 of additional parts.

Software: We wanted to broadcast the messages created by Emlo, and settled on Twitter as the platform due to its familiarity and easy/robust API. The only option we found for reverse-geocoding coordinates was the Google Maps API, which won by default. We also wanted to add some additional functionality, such as customizable emotion labels, customizable location labels, and "listen" mode where Emlo receives data from other virtual Emlo devices. Due to my existing familiarity with Django, we decided the most robust and simple solution was to use Google App Engine as the middle layer handling these tasks.

Final Project Components (Creator in parenthesis):
  • Sensor Housing - The exterior surface of Emlo (Jessica, Alison)
  • Arduino Sensors and Code - The guts and brains of Emlo, built using an Arduino Duemilanove with flex sensor, force sensor, and photo sensor (Guts: Alison, Brains: Alison, Ian)
  • Python glue code - Local machine code for translating the Arduino serial input into HTTP Requests for the App Engine Back End. (Ian)
  • App Engine Back End - Used to store user messages, store per-user associations between gestures/locations and plain English names, reverse code geodata using the Google Maps API, call the Twitter API, and host the administrator front end. (Ian)
  • App Engine Front End - Administrator controls for managing descriptions of locations and gestures to be used in posted tweets. Also includes tool for generating fake tweets for demonstration purposes. (Ian)
  • Visual Assets - The project poster, clay prototypes, and digital renderings of Emlo (Jessica).
