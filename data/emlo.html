Emlo the "tangible broadcast buddy," is a prototype device for broadcasting your current emotion from a given location. User gestures are translated into a corresponding emotion, which is then posted (in combination with the user's location) to Twitter. Emlo was a class final project in collaboration with Alison Meir and Jessica Voytek.

Emlo was born out of an interest to explore gestural expression of emotion. After brainstorming different methods of exploring this question, we settled on the idea of a digital pet able to relay how its master is feeling. To inform our design, we started with a small number of exploratory interviews, and used the output to produce a close-ended survey.

Interviews

Six participants were individually interviewed and asked to model each of eight basic emotions using provided materials with different physical properties (below). The gestures and materials used were recorded and clustered. Ten distinct gestures were identified, which served as the basis for a follow-on survey.

Survey

We created a clay prototype and used it to demonstrate each gesture identified in the exploratory interviews in a series of short video clips. We incorporated these in an online survey asking users to map each of the eight emotions to one gesture (some gestures were left over). The 35 responses showed strongest agreement in gesture/emotion mappings for anger, sadness, joy, and surprise.

Hardware:

The class focused on the Arduino Duemilanove microcontroller, which we used as the base of our stack. We then took the top four emotion/gesture pairs from our research and figured out how to model them with off-the-shelf sensors.

Software:

We chose Twitter for a broadcast platform to take advantage of its easy and robust API. Reverse-geocoding coordinates was accomplished using the Google Maps API. We also wanted some additional functionality, including user-customizable labels for emotions and locations, and a "listen" mode where Emlo receives and relays data from other virtual Emlo devices. Google App Engine served as the middle layer handling these tasks.

Emlo Components and Responsibilities (lead developer in bold):
  • Sensor Housing - Emlo's body (Jessica, Alison)
  • Arduino Sensors and Code - The guts and brains of Emlo, built using an Arduino Duemilanove with flex sensor, force sensor, and photo sensor (Guts: Alison, Brains: Alison, Ian)
  • Python glue code - Local machine code for translating the Arduino serial input into HTTP Requests for the App Engine Back End. (Ian)
  • App Engine Back End - Used to store user messages, store per-user associations between gestures/locations and plain English names, reverse code geodata using the Google Maps API, call the Twitter API, and host the administrator front end. (Ian)
  • App Engine Front End - Administrator controls for managing descriptions of locations and gestures to be used in posted tweets. Also includes tool for generating fake tweets for demonstration purposes. (Ian)
  • Visual Assets - The project poster, clay prototypes, and digital renderings of Emlo (Jessica).

Emlo was well received by the class and school community; The final grade for the project was an 'A+', Emlo was featured on the front page of the I School website, and asked to re-appear at several community I School recruiting events.
